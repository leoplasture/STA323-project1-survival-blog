## 3. Background

This post summarizes my Project 1 Q2 work, including survival analysis on the IBM Telco Churn dataset and a Text-to-SQL error analysis task in MySQL.

## 4. Data and Cohort Definition

I used the IBM Telco churn data and built a survival-analysis cohort with the following filters:

- Contract is Month-to-month
- InternetService is not No

The event variable is churn (1 for Yes, 0 for No), and the duration variable is tenure.

## 5. Kaplan-Meier Analysis

I first applied Kaplan-Meier estimation to model customer survival over tenure.

Key result:

- Cohort median survival time: 34 months

Group comparison highlights:

- Gender difference was not statistically significant (p = 0.1533)
- OnlineSecurity showed a very strong survival difference (p ≈ 1.19e-32)
- Payment methods showed strong differences, especially between automatic payment and electronic check groups

## 6. Cox Proportional Hazards Model

I fitted a Cox PH model on selected one-hot encoded features.

Key outputs:

- Number of observations: 3351
- Events observed: 1556
- Concordance: 0.64

Important effects (all significant):

- dependents_Yes: exp(coef)=0.72
- internetService_DSL: exp(coef)=0.80
- onlineBackup_Yes: exp(coef)=0.46
- techSupport_Yes: exp(coef)=0.53

Interpretation: these features are associated with lower churn hazard.

PH assumption check indicated violations for several variables, so the baseline Cox model is informative but not fully PH-compliant.

## 7. AFT Model (Log-Logistic)

I also fitted a LogLogistic AFT model.

Key outputs:

- Median survival time (model output): 135.51
- Concordance: 0.73
- AIC: 13698.72

Compared to Cox, the AFT model achieved better ranking performance in this run.

## 8. Text-to-SQL Task in MySQL

After importing data into MySQL, I tested LLM-generated SQL and documented three failure cases:

1. Missing required cohort filters in the WHERE clause
2. Using unsupported function MEDIAN in MySQL
3. Wrong denominator in grouped ratio calculation

For each case, I recorded:

- database schema
- user question
- LLM-generated wrong SQL
- corrected SQL
- failure reason

## 10. Conclusion

Main findings:

- Survival behavior differs strongly by security-related services and payment behavior
- Cox model gives interpretable hazard effects but violates PH assumptions for some predictors
- AFT model provided stronger ranking performance in this experiment
- LLM Text-to-SQL is useful for drafting but still needs strict manual validation
